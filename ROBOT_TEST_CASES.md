## 1. 테스트 환경 표준화 (Standard Environment)

모든 테스트는 아래의 표준 환경을 기본으로 하며, 별도 명시가 없는 한 이 조건을 유지한다.

*   **바닥 상태 (Floor)**: 실내 나무 마루(Wooden Floor), 단차 및 미끄러짐 요소 없음.
*   **네트워크 (Network)**: Wi-Fi 5(802.11ac) 기반 로컬 네트워크
*   **기본 로봇 상태 (Robot Status)**: 
    *   배터리 잔량 50% 이상.
    *   카메라 렌즈 이물질 제거 확인.

## 2. 변수 통제 테스트 (Level 1~4)

각 테스트 레벨은 지정된 **독립 변수(Independent Variable)** 외의 모든 환경 요소를 동일하게 통제한다.

### 1. 거리 변화 (Distance Variation)
*   **공통 조건**: 장애물 없음, 조도 형광등(Normal), 경로 직선, 프롬프트-목표물 일치 ("컵으로 이동" -> `cup`).

| ID | 테스트 케이스 | 독립 변수 (거리) | 통과 기준 (Success Criteria) |
| :--- | :--- | :--- | :--- |
| 1-1 | Easy | 30cm | 목표물 앞 5cm 이내 정지 및 탐지 유지 |
| 1-2 | Normal | 60cm | 경로 이탈 없이 정속 주행 및 정지 |
| 1-3 | Hard | 150cm | 원거리 객체 탐지(Confidence > 0.5) 및 안정적 도달 |

### 2. 장애물 복잡도 (Obstacle Complexity)
*   **공통 조건**: 거리 1.0m, 조도 형광등, 우회 경로 필수, 프롬프트-목표물 일치 ("의자로 가줘" -> `chair`).

| ID  | 테스트 케이스 | 독립 변수 (장애물)      | 통과 기준 (Success Criteria)       |
| :-- | :------ | :--------------- | :----------------------------- |
| 2-1 | Easy    | 장애물 작음 (종이컵 등)   | 장애물 접촉 없이 최소 회전 반경으로 우회        |
| 2-2 | Normal  | 장애물 큼 (박스, 가구 등) | VLM이 우회 경로를 판단하고 안전 거리 확보 후 도달 |

### 3. 동적/맥락 해석 (Context & Dynamic)
*   **공통 조건**: 거리 1.5m 이상, 조도 형광등.

| ID  | 테스트 케이스 | 독립 변수 (상황)               | 목표 객체      |
| :-- | :------ | :----------------------- | :--------- |
| 3-1 | Easy    | 내 핸드폰이 어디 갔지?            | cell phone |
| 3-2 | Normal  | 목이 너무 말라서 그런데 마실 것 좀 있나? | bottle     |
| 3-3 | Hard    | 창밖이 갑자기 어두워지고 빗방울이 떨어지네. | umbrella   |

## 3. 기술 통계 기반 특수 테스트 (Metric-Specific)

현재 시스템의 주요 결함 이슈를 해결하기 위한 집중 테스트 항목이다.
*   **공통 통제 항목**: 프롬프트 **"물병 앞으로 가줘"** 및 목표 객체 **`bottle`** 고정.

### 3-1. 네트워크 지연 대응 (Network Latency)
*   **독립 변수**: RTT(Round Trip Time) 4,000ms 강제 지연 시뮬레이션.
*   **목표**: 제어 명령 전달 지연 상황에서 객체 인식 실패나 충돌 발생 여부 확인.
*   **통과 기준**: 지연 발생 시 로봇이 속도를 감속하거나 정지하여 `Safety First` 모드로 진입하는지 확인.

### 3-2. 뎁스 정보 유실 복구 (Depth Loss Recovery)
*   **독립 변수**: `target_depth_unavailable` 유도 (가림막으로 센서 순간 차단).
*   **목표**: 뎁스 유실 시 시스템의 비정상 종료 방지 및 재확보 후 주행 재개 능력 평가.
*   **통과 기준**: 뎁스 유실 알람 발생 후, 가림막 제거 시 2초 이내에 거리 정보를 복구하고 목표물 도달 성공.

### 3-3. 저조도 신뢰도 유지 (Low Confidence Management)
*   **독립 변수**: 저조도 (20lx 이하) 환경 조성.
*   **목표**: Low Confidence(0.132 미만) 상황에서의 오탐지(False Positive) 방지 성능 평가.
*   **통과 기준**: Confidence가 임계치(0.132) 미만일 경우 무리한 주행 대신 "객체 확인 불가" 피드백 또는 제자리 회전을 통한 재탐색 수행.

## 4. 평가 지표 (Evaluation Metrics)

| 지표명 (Metric)                | 설명                           | 목표치 (Target)    |
| :-------------------------- | :--------------------------- | :-------------- |
| **PSR (Path Success Rate)** | 전체 시도 중 목표 지점에 안전하게 도달한 비율   | 90% 이상          |
| **RTT (Round Trip Time)**   | 명령 하사부터 로봇 반응 시작까지의 소요 시간    | 500ms 이하 (정상)   |
| **Stop Distance Error**     | 목표물 앞 목표 정지 거리와 실제 정지 거리의 차이 | ±3cm 이내         |
| **Recovery Success Rate**   | 뎁스 유실 또는 네트워크 단절 후 복구 성공 비율  | 95% 이상          |
| **Confidence Level**        | YOLOv8 객체 탐지 신뢰도 점수 평균       | 0.75 이상 (표준 환경) |
